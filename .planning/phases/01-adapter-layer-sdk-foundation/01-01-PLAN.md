  name: string;
  description: string;

  label: string;

  | { type: "message_update"; message: AgentMessage; assistantMessageEvent: AssistantMessageEvent }
  | { type: "message_end"; message: AgentMessage }
  | { type: "tool_execution_start"; toolCallId: string; toolName: string; args: any }
  | { type: "tool_execution_update"; toolCallId: string; toolName: string; args: any; partialResult: any }

3. `bridgeToolToCopilot(tool: AgentTool, context: ToolBridgeContext)` function:
   - Takes an `AgentTool` and a `ToolBridgeContext` (providing `signal?: AbortSignal`)
   - Returns the result of `defineTool(tool.name, { description: tool.description, parameters: tool.parameters, handler })`.
   - The `handler` async function: receives `args`, generates a `toolCallId` (e.g. `crypto.randomUUID()`), calls `tool.execute(toolCallId, args, context.signal)`, extracts text content from the `AgentToolResult.content` array and returns it as a string (concatenate all `TextContent` blocks).
   - Does NOT modify tool business logic — pure mechanical bridge per D-06.

4. `ToolBridgeContext` interface:
   - `signal?: AbortSignal` — abort signal passed through to tool execute

5. `bridgeAllTools(tools: AgentTool[], context: ToolBridgeContext)` convenience function:
   - Returns `tools.map(t => bridgeToolToCopilot(t, context))`.

Export `bridgeToolToCopilot`, `bridgeAllTools`, `ToolBridgeContext`.

Update `index.ts` to re-export these.

Note: The SDK's `defineTool` accepts JSON Schema for parameters. TypeBox schemas from `@sinclair/typebox` are JSON Schema compatible — no conversion needed. The `handler` wraps the existing `execute()` call with minimal glue.

Create `packages/pi-coding-agent/src/core/backends/event-translator.ts` with:

1. Import `AgentEvent` from `@gsd/pi-agent-core`.

2. Define `CopilotSessionEvent` type representing the SDK event shape:
   ```typescript
   interface CopilotSessionEvent {
     type: string;
     data?: Record<string, any>;
   }
   ```

3. `translateCopilotEvent(event: CopilotSessionEvent): AgentEvent | null` function with a switch on `event.type`:

   - `"assistant.turn_start"` → `{ type: "turn_start" }`
   - `"assistant.message"` → `{ type: "message_end", message: { role: "assistant", content: [{ type: "text", text: event.data.content }] } }` (construct minimal AgentMessage)
   - `"assistant.message_delta"` → `{ type: "message_update", message: { role: "assistant", content: [{ type: "text", text: event.data.deltaContent }] }, assistantMessageEvent: { deltaContent: event.data.deltaContent } }` (for streaming)
   - `"tool.execution_start"` → `{ type: "tool_execution_start", toolCallId: event.data.toolCallId, toolName: event.data.toolName, args: event.data.arguments ?? {} }`
   - `"tool.execution_complete"` → `{ type: "tool_execution_end", toolCallId: event.data.toolCallId, toolName: event.data.toolName ?? "", result: event.data.result ?? event.data.error, isError: !event.data.success }`
   - `"session.idle"` → return null (handled at session level, not as AgentEvent)
   - `"session.error"` → return null (handled via promise rejection, not as AgentEvent)
   - `"assistant.usage"` → return null (Phase 2 telemetry concern)
   - default → return null (unknown events silently ignored)

4. `isSessionIdle(event: CopilotSessionEvent): boolean` — returns true if `event.type === "session.idle"`.

5. `isSessionError(event: CopilotSessionEvent): boolean` — returns true if `event.type === "session.error"`.

Export `translateCopilotEvent`, `CopilotSessionEvent`, `isSessionIdle`, `isSessionError`.

Update `index.ts` to re-export these.

Note: The AgentMessage construction is deliberately minimal — only the fields that downstream event consumers (RPC mode, bridge service) actually read. The `agent_start` and `agent_end` events are emitted by the backend session handle wrapper, not from SDK event translation.
