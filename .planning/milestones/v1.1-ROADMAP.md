## Milestone Summary

**Key Accomplishments:**
1. Execute/verify workflows ported to Copilot SDK backend with full CLI dispatch and test coverage (EXEC-01)
2. Full autonomous orchestration migrated — per-unit stage/tool config threaded through dispatch → runUnit → newSession (EXEC-02)
3. Roadmap/requirements management commands wired through Copilot backend at 0.33× accounting tier (FLOW-01)
4. Free-tier model downgrade (`suggestDowngrade()`) wired into `CopilotSessionBackend` — automatic 0× fallback with stderr telemetry (FLOW-02)
5. BYOK fallback (`_applyByokIfExhausted()`) wired into session creation and resume paths — activates on quota exhaustion (FLOW-03)
6. Telemetry consumers wired: `getDowngrades()` and `getByokActivations()` feed into `formatPremiumSummary` at session teardown
7. All v1.1 phases (08–12) achieved Nyquist compliance with accurate post-execution VALIDATION.md files

**Gap Closure Phases Added:**
- Phase 13: Fixed `setSettingsManager` wiring bug in `sdk.ts` (BYOK was silently bypassed on every session)
- Phase 14: Wired orphaned telemetry accessors into `formatPremiumSummary` callers
- Phase 15: Created/updated all VALIDATION.md files from draft-state to Nyquist-compliant

**Key Decisions:**
- `codingTools` for execute (full read/bash/edit/write per D-07); `readOnlyTools` for verify (no write/edit per D-08)
- `stage: 'execute-task'` (not `'execute-phase'`) used in createAgentSession to match existing STAGE_TIER_MAP key
- Source-shape tests (readFileSync pattern assertions) used throughout for routing/tool-profile contracts — avoids SDK auth dependencies
- `activeToolNames` applied AFTER `_buildRuntime` in newSession to prevent extension rebuild from overriding per-unit tool restriction
- `suggestDowngrade()` is pure — safe to call at session-creation boundary, never mid-send (D-03)
- `DEFAULT_FREE_TIER_FALLBACK enabled:true` — opt-out model so fallback is active by default
- BYOK + downgrade checks at session creation boundary only (D-03/D-07/D-08)

**Technical Debt Carried Forward:**
- `UnitSessionConfig.stage` field populated by auto-dispatch but not consumed by extension `newSession()` — stage cannot thread through to the backend session in the extension path; deferred to future extension integration work
- `formatPremiumSummary` telemetry only reaches `process.stderr` — no structured JSON log or persistent telemetry store; deferred to v1.2+
- Phase 11/12 fallbacks (downgrade + BYOK) do not apply to the auto-mode extension session plane (`s.cmdCtx.newSession`); auto-mode uses VS Code extension sessions, not `createAgentSession()`; future work needed to bridge accounting to extension sessions
