<objective>
Propagate workflow stage metadata end-to-end from planning workflow entry points through the backend session send contract into accounting tracker records, replacing the hardcoded `stage = "unknown"` fallback with explicit stage attribution.

Purpose: Closes the stage metadata integration gap identified in the v1.0 milestone audit. Without this, all Copilot backend accounting records are attributed to "unknown" stage at standard tier, defeating the multiplier-aware cost tracking built in Phase 2.

Output: Modified send contract with optional stage metadata, accounting handle using configured stage, workflow code supplying canonical stage values, and integration tests proving the propagation chain.
</objective>
