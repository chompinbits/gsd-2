Create `request-tracker.ts`:

- `RequestTracker` class (instantiated per session, not global):
  - Constructor takes `sessionId: string` and `budgetLimit: number`.
  - `record(model: string, stage: string, tier: MultiplierTier)`: creates a `RequestRecord`, pushes to internal array, increments `_totalPremiumRequests` by `getMultiplierValue(tier)`. Per D-03, each call = one completion.
  - `getState(): BudgetState`: returns `{ totalPremiumRequests, records, budgetLimit, percentUsed }` where `percentUsed = (totalPremiumRequests / budgetLimit) * 100`. Handle budgetLimit=0 as 0%.
  - `getSummary(): PremiumRequestSummary`: aggregates records by stage, returns per-stage count and premium cost per D-13.
  - `reset()`: clears records and totals (for session reset scenarios).
  - `toJSON()`: returns serializable representation for persistence per D-04/D-12.
  - `static fromJSON(data: unknown, sessionId: string): RequestTracker`: restores from persisted data.
- Import types from `./types.js` and multiplier functions from `./multipliers.js`.

Create `budget-guard.ts`:

- `BudgetWarning` interface: `{ type: "warning"; message: string; percentUsed: number }`.
- `BudgetExceededError` class extending `Error`: includes `used: number`, `limit: number`, `percentUsed: number`. Message per D-10: "Premium request budget exceeded: {used}/{limit} requests used ({percentUsed}%). Adjust thresholds with `gsd settings` or use `/gsd-set-profile budget` to reduce model tier."
- `BudgetGuard` class:
  - Constructor takes `config: AccountingConfig` and `tracker: RequestTracker`.
  - `check(estimatedCost: number): "ok" | BudgetWarning`: Pre-send check per D-09.
    1. Project new total = `tracker.getState().totalPremiumRequests + estimatedCost`.
    2. Calculate `percentUsed = (projected / config.budgetLimit) * 100`.
    3. If `percentUsed >= 100` and `config.hardStop`: throw `BudgetExceededError`.
    4. If `percentUsed >= 100` and not `config.hardStop`: return warning (soft limit per D-11).
    5. If `percentUsed >= config.warnThreshold * 100`: return `BudgetWarning`.
    6. Otherwise: return `"ok"`.
  - Handle budgetLimit=0 as "unlimited" (always return "ok") to allow disabling budget enforcement.
  </action>
