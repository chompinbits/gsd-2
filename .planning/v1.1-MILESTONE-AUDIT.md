### GAP-2: FLOW-03 — BYOK never activates in production (Integration)

`sdk.ts` creates `CopilotSessionBackend` and wires accounting config but never calls `setSettingsManager()`. The `_applyByokIfExhausted()` method guards on `this._settingsManager` being set:

```typescript
if (!this.accountingConfig || !this._currentTracker || !this._settingsManager) {
  return { config, byokActive: false };  // always exits here
}
```

**Effect:** BYOK silently bypasses on every `createAgentSession()` call — all CLI workflow paths (execute-phase, verify-work, new-project, new-milestone, add-phase, remove-phase, progress) are affected.

**Fix:** In `packages/pi-coding-agent/src/core/sdk.ts`, add after the `setAccountingConfig` call:
```typescript
copilotBackend.setSettingsManager(settingsManager);
```

**Affects:** FLOW-03 ("User can use BYOK fallback when premium quota is exhausted")
